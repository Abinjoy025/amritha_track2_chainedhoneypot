⚙️ How It Works

1️⃣ Deception Layer

A convincing fake admin portal is exposed publicly. Attackers see a login page indistinguishable from a real admin panel. Every interaction is silently captured — username, password, headers, raw payload, User-Agent, cookies, and query parameters.


Flask X-Forwarded-For SHA256 payload hash

2️⃣ AI Classification

A Random Forest model trained on the NSL-KDD dataset (125,000+ records, 22 attack types) classifies each event in real-time. Three modes: REAL (Scapy packet features), HEURISTIC (rule-based), and RULES_ONLY (no model). Heuristic rules detect DoS, Probe, R2L, U2R patterns.


Random Forest NSL-KDD 99.94% accuracy

3️⃣ Infrastructure Intelligence

Every attacker IP is profiled using AbuseIPDB (abuse score), IPinfo (ASN, org, city), VirusTotal (malicious detections), and a live Tor exit node list. VPN/datacenter detection via ASN keywords classifies whether the attacker is a script kiddie, botnet, or targeted actor.


AbuseIPDB IPinfo VirusTotal Tor check

4️⃣ Behavioral Fingerprinting

VPN hides location but not behavior. Each IP builds a behavioral profile: timing patterns (machine-speed vs. human), credential strategies (spray / stuffing / targeted), tool signatures in User-Agent, and a behavioral fingerprint hash that links multiple IPs to the same actor.


Timing analysis Fingerprint hash Actor correlation

5️⃣ Campaign Tracking

Attacks are grouped into campaigns using behavioral fingerprints, MITRE ATT&CK TTP clustering, and malware family signatures. A single campaign can span dozens of rotating IPs — you track the actor, not just the address. Follows MITRE ATT&CK framework thinking.


MITRE ATT&CK TTP clustering Multi-IP actor

6️⃣ Immutable Evidence Chain

Confirmed attack evidence (including IP intel, behavioral profile, payload fingerprint, and full log entry) is stored on IPFS via Pinata for content-addressed permanent storage, with the CID hash anchored on Ethereum Sepolia testnet for tamper-proof timestamping.

🔧 Problems Faced & How We Solved Them

❌ Problem 1: AI Always Returned "Benign" for All Attacks

The Random Forest model was trained on NSL-KDD network features (41 packet-level features). Feeding it zero-vectors built from credential data always predicted "benign" because no actual network patterns were present.

✅ Solution: Rewrote the predictor with 3 modes. When packet capture data is unavailable (no root/sudo), the HEURISTIC mode uses rule-based classification with U2R/R2L/brute-force pattern detection instead of misusing the RF model.

❌ Problem 2: Services Crashing in Background

Running Flask with debug=True spawns a reloader process that requires a TTY. When backgrounded, it killed itself immediately.

✅ Solution: Set debug=False, use_reloader=False in all Flask apps. Each service runs in its own isolated background terminal.

❌ Problem 3: VPN Makes Geolocation Useless

Modern attackers route through VPN, Tor, or cloud VPS, making country-based attribution meaningless. Traditional honeypots only store IP + country.

✅ Solution: Layer 2-5 intelligence: ASN/org detection for VPN providers, behavioral fingerprinting (attack timing, credential patterns, tool signatures), cross-IP campaign correlation — attacker behavior is consistent even through VPN.

❌ Problem 4: Static Model Can't Learn New Attack Patterns

NSL-KDD is a 2009 dataset. New attack tools, botnets, and techniques that didn't exist then would be classified incorrectly.

✅ Solution: Online Learner module appends every confirmed attack to a local CSV and retrains the RF model every 20 samples, giving the model 3× weight on new honeypot data vs. original NSL-KDD data.

❌ Problem 5: Evidence Can Be Deleted / Tampered

Log files on a compromised server can be wiped. There's no way to prove an attack happened after the fact if logs are deleted.

✅ Solution: Every confirmed attack is stored on IPFS (content-addressed — the CID is a hash of the data, so it cannot be modified) and the CID is anchored on Ethereum blockchain with a timestamp, creating a tamper-proof evidence chain that survives server compromise.

❌ Problem 6: DoS / Port-Scan Attacks Show as Benign in HEURISTIC Mode

DoS and Probe attacks don't have credential patterns — they flood packets. Without raw packet access (requires root), heuristic rules can't detect them.

✅ Solution: Run controller with sudo to enable REAL mode — Scapy packet capture extracts actual NSL-KDD network features (connection rate, error rate, service distribution) which the RF model can correctly classify.

🗂️ NSL-KDD Attack Taxonomy (What We Detect)

⚡ DoS — Denial of Service

Overwhelm a target with traffic to make it unavailable.
Subtypes: backland neptunepod smurfteardrop

Detection: High connection count, high error rate, SYN flood patterns. Requires REAL mode (packet features).

🔍 Probe — Reconnaissance

Scan a network to find vulnerabilities before attacking.
Subtypes: ipsweepnmap portsweepsatan

Detection: Many different destination ports, low bytes per connection, systematic IP sweep patterns.

👤 R2L — Remote to Local

Attacker gains local access by exploiting a vulnerability.
Subtypes: ftp_writeguess_passwd imapphf spywarezclient

Detection: Repeated failed logins, known username lists, brute-force credential patterns (works in HEURISTIC mode).

⚠️ U2R — User to Root

Attacker escalates privileges to gain root/admin control.
Subtypes: buffer_overflowloadmodule perlrootkit

Detection: Root shell spawn, privilege escalation commands in payload, buffer overflow attempts, /etc/shadow access (works in HEURISTIC mode).
